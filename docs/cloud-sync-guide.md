## 功能概述

AlgerClipboard 支持将剪贴板历史记录同步到云端，实现多设备之间的数据共享。主要特性：

- 支持 WebDAV、Google Drive、OneDrive 三种云存储
- 可选端到端加密（AES-256-GCM + Argon2id）
- 增量同步，仅传输变更内容
- 自动冲突检测与解决
- 灵活的同步频率设置（实时 / 定时 / 手动）

## 支持的云存储

| 存储类型 | 认证方式 | 适用场景 |
|---------|---------|---------|
| WebDAV | 用户名 + 密码 | 自建 NAS（群晖、NextCloud 等）、坚果云 |
| Google Drive | OAuth 2.0 | Google 生态用户 |
| OneDrive | OAuth 2.0 | Microsoft 365 / Windows 用户 |

## WebDAV 配置

WebDAV 是最简单的同步方式，适合自建服务或国内云存储。

### 准备工作

确保你有一个支持 WebDAV 协议的服务，常见选项：

- **坚果云** — 免费支持 WebDAV，国内速度快
- **NextCloud** — 开源自建网盘
- **群晖 NAS** — 内置 WebDAV Server 套件
- **AList** — 支持多种网盘的 WebDAV 转接

### 配置步骤

1. 打开 **设置** → **同步** 标签页
2. 点击 **添加云存储**
3. 选择 **WebDAV** 作为存储类型
4. 填写以下信息：

| 字段 | 说明 | 示例 |
|-----|------|-----|
| 服务器地址 | WebDAV 服务完整 URL | `https://dav.jianguoyun.com/dav/` |
| 用户名 | 登录账号 | `user@example.com` |
| 密码 | 登录密码或应用专用密码 | `app-password-here` |

5. 点击 **测试连接** 确认配置正确
6. 设置 **同步频率**
7. 点击 **保存**

### 坚果云配置示例

1. 登录 [坚果云](https://www.jianguoyun.com)
2. 进入 **账户信息** → **安全选项** → **第三方应用管理**
3. 点击 **添加应用**，输入名称 `AlgerClipboard`
4. 复制生成的 **应用密码**
5. 在 AlgerClipboard 中配置：
   - 服务器地址：`https://dav.jianguoyun.com/dav/`
   - 用户名：你的坚果云邮箱
   - 密码：上一步生成的应用密码

### NextCloud 配置示例

- 服务器地址：`https://your-domain.com/remote.php/dav/files/username/`
- 用户名：NextCloud 用户名
- 密码：NextCloud 密码或应用密码（建议在 设置 → 安全 中生成应用密码）

## Google Drive 配置

使用 Google Drive 需要先创建 OAuth 凭证。

### 准备工作

1. 访问 [Google Cloud Console](https://console.cloud.google.com/)
2. 创建新项目（或选择已有项目）
3. 启用 **Google Drive API**：
   - 进入 **API 和服务** → **库**
   - 搜索 `Google Drive API` 并启用
4. 创建 OAuth 凭证：
   - 进入 **API 和服务** → **凭证**
   - 点击 **创建凭证** → **OAuth 客户端 ID**
   - 应用类型选择 **桌面应用**
   - 记录 **Client ID** 和 **Client Secret**
5. 配置 OAuth 同意屏幕：
   - 进入 **OAuth 同意屏幕**
   - 用户类型选择 **外部**
   - 添加测试用户（你的 Google 邮箱）

### 配置步骤

1. 打开 **设置** → **同步** 标签页
2. 点击 **添加云存储**
3. 选择 **Google Drive**
4. 填写 **Client ID** 和 **Client Secret**
5. 点击 **授权**，浏览器将自动打开 Google 登录页面
6. 登录 Google 账号并授权 AlgerClipboard 访问你的 Drive 文件
7. 授权成功后窗口会显示 "Authorization successful!"
8. 回到 AlgerClipboard，设置同步频率并保存

### 数据存储位置

同步数据存储在你的 Google Drive 根目录下的 `AlgerClipboard` 文件夹中。

### 权限说明

AlgerClipboard 仅申请 `drive.file` 权限，即只能访问由本应用创建的文件，**无法读取你 Drive 中的其他文件**。

## OneDrive 配置

使用 OneDrive 需要创建 Microsoft 应用注册。

### 准备工作

1. 访问 [Azure 应用注册](https://portal.azure.com/#blade/Microsoft_AAD_RegisteredApps/ApplicationsListBlade)
2. 点击 **新注册**：
   - 名称：`AlgerClipboard`
   - 支持的帐户类型：**个人 Microsoft 帐户**
   - 重定向 URI：选择 **Web**，地址填 `http://127.0.0.1/callback`（端口会动态替换）
3. 记录 **应用程序(客户端) ID**

### 配置步骤

1. 打开 **设置** → **同步** 标签页
2. 点击 **添加云存储**
3. 选择 **OneDrive**
4. 填写 **Client ID**（应用程序 ID）
5. 点击 **授权**，浏览器将自动打开 Microsoft 登录页面
6. 登录并授权
7. 设置同步频率并保存

### 数据存储位置

同步数据存储在你的 OneDrive 根目录下的 `AlgerClipboard` 文件夹中。

### 权限说明

AlgerClipboard 申请 `Files.ReadWrite.All` 和 `offline_access` 权限，用于读写文件和获取刷新令牌。

## 同步频率

在添加云存储时，可以选择以下同步频率：

| 模式 | 说明 | 适用场景 |
|-----|------|---------|
| **实时** | 剪贴板变更后立即同步（5 秒去抖） | 需要即时同步的场景 |
| **定时** | 按设定间隔自动同步（1 ~ 1440 分钟） | 平衡性能与时效性 |
| **手动** | 仅在手动点击"立即同步"时触发 | 流量敏感或偶尔同步 |

### 建议

- 日常使用推荐 **定时 15 分钟**
- 局域网内 NAS 推荐 **实时**
- 移动网络或流量有限时推荐 **手动**

## 端到端加密

AlgerClipboard 支持对同步数据进行端到端加密，确保云端存储的数据无法被第三方（包括云存储提供商）读取。

### 加密方案

| 组件 | 算法 |
|-----|------|
| 密钥派生 | Argon2id（从用户密码推导 256 位密钥） |
| 数据加密 | AES-256-GCM（认证加密） |
| 随机数 | 每次加密生成 12 字节随机 Nonce |
| 盐值 | 首次启用时生成 16 字节随机 Salt |

### 启用加密

1. 在添加或编辑云存储账户时，开启 **端到端加密** 开关
2. 输入 **加密密码**（建议 12 位以上，包含大小写字母和数字）
3. 保存设置

### 重要提示

- **请牢记加密密码**，密码丢失将无法恢复云端数据
- 所有设备必须使用 **相同的加密密码** 才能正常同步
- 加密密码修改后，需要在所有设备上同步更新
- 加密会略微增加同步数据大小和同步耗时

### 加密范围

启用加密后，以下内容将被加密：

- 剪贴板条目元数据（JSON）
- 图片和文件的二进制数据
- 同步清单（manifest）

## 同步机制

### 数据流向

```
设备 A                     云端                      设备 B
  │                         │                         │
  ├── 推送本地变更 ────────→ │                         │
  │                         │ ←──────── 推送本地变更 ──┤
  │                         │                         │
  ├── 拉取远程变更 ←──────── │                         │
  │                         │ ────────→ 拉取远程变更 ──┤
  │                         │                         │
```

### 同步流程

1. **初始化**：在云端创建目录结构
   ```
   AlgerClipboard/
   ├── manifest.json          ← 同步清单（记录所有条目的版本）
   ├── entries/               ← 条目元数据
   │   ├── {id-1}.json
   │   └── {id-2}.json
   └── blobs/                 ← 二进制文件（图片等）
       ├── {hash-1}.bin
       └── {hash-2}.bin
   ```

2. **推送（Push）**：将本地新增/修改的条目上传到云端

3. **拉取（Pull）**：下载云端的新条目或更新

4. **更新清单**：同步完成后更新远端 manifest.json

### 增量同步

每个剪贴板条目维护一个 `sync_version` 版本号。同步时只传输版本号大于上次同步版本的条目，避免重复传输已同步的数据。

### 状态指示

同步状态通过标题栏图标和条目徽标展示：

| 图标 | 含义 |
|-----|------|
| ☁️ 绿色 | 已同步 |
| ⬆️ 蓝色 | 等待上传 |
| ⚠️ 橙色 | 存在冲突 |
| 🔄 旋转 | 正在同步 |
| ❌ 红色 | 同步失败 |

## 冲突处理

当同一条目在多个设备上被修改时，会产生同步冲突。

### 冲突检测条件

同时满足以下条件时触发冲突：
- 本地版本的内容哈希（content_hash）与远端不一致
- 本地版本在上次同步后发生过修改

### 冲突解决方式

| 选项 | 说明 |
|-----|------|
| **保留本地** | 以本地版本为准，下次同步时覆盖远端 |
| **保留远程** | 以远端版本为准，下次同步时拉取覆盖 |
| **保留两者** | 同时保留两个版本，创建冲突副本 |

冲突条目在列表中会显示橙色警告图标，右键菜单中可选择解决方式。

## 常见问题

### WebDAV 连接失败

**可能原因：**
- 服务器地址末尾缺少 `/`
- 使用了登录密码而非应用专用密码（如坚果云）
- 服务器 SSL 证书问题
- 防火墙或代理阻止了连接

**排查步骤：**
1. 确认服务器地址格式正确（以 `https://` 开头，以 `/` 结尾）
2. 在浏览器中访问 WebDAV 地址，确认可以正常打开
3. 使用应用专用密码而非账户密码
4. 检查网络连接和防火墙设置

### Google Drive / OneDrive 授权失败

**可能原因：**
- Client ID 或 Client Secret 填写错误
- OAuth 同意屏幕未配置
- 未将自己的邮箱添加为测试用户
- 重定向 URI 配置不正确

**排查步骤：**
1. 检查 Client ID 是否完整复制（不要有多余空格）
2. 确认 OAuth 同意屏幕已发布或已添加测试用户
3. 重新创建 OAuth 凭证并重试

### 同步后数据未出现在其他设备

**可能原因：**
- 另一台设备未配置同步或同步频率设置为手动
- 加密密码不一致
- 网络问题导致同步未完成

**排查步骤：**
1. 检查两台设备是否使用相同的云存储账户
2. 在另一台设备上点击 **立即同步**
3. 确认两台设备的加密密码一致
4. 查看标题栏同步状态图标是否显示错误

### 加密密码忘记了怎么办

遗憾的是，加密密码无法找回。你需要：

1. 在所有设备上删除同步账户
2. 手动删除云端 `AlgerClipboard` 文件夹
3. 使用新密码重新配置同步

### 如何迁移到其他云存储

1. 在旧的同步账户上点击 **立即同步**，确保数据已完全同步
2. 添加新的云存储账户
3. 点击新账户的 **立即同步**
4. 确认数据同步完成后，删除旧账户

### 同步占用多少存储空间

- 每条文本记录约占 1~5 KB
- 图片记录取决于原始图片大小
- 启用加密后数据量会略微增加（约 10~20%）
- 1000 条纯文本记录大约占 2~5 MB

## 技术规格

| 项目 | 规格 |
|-----|------|
| 同步协议 | HTTPS (WebDAV / REST API) |
| 加密算法 | AES-256-GCM |
| 密钥派生 | Argon2id |
| 冲突检测 | 基于内容哈希 + 版本号 |
| 数据格式 | JSON（元数据）+ Binary（文件） |
| 最小同步间隔 | 1 分钟 |
| OAuth 回调 | 本地 HTTP 服务器（127.0.0.1 随机端口） |
