# 搜索指南

AlgerClipboard 内置基于 SQLite FTS5 的全文搜索引擎，支持多字段搜索、拼音匹配、正则表达式、时间范围筛选和搜索历史。

## 基本搜索

在搜索栏输入关键词，将同时搜索以下字段：

- **文本内容** — 剪贴板文本、富文本
- **AI 摘要** — 自动生成的 AI 摘要
- **文件名** — 复制的文件名称
- **来源应用** — 内容复制来源的应用程序
- **标签** — 用户添加的标签
- **拼音** — 中文拼音（全拼 + 首字母）
- **图片 OCR 文字** — 通过 OCR 从图片中提取的文字

### 多关键词搜索

用空格分隔多个关键词，所有关键词必须同时匹配（AND 逻辑）：

```
hello world       → 同时包含 "hello" 和 "world" 的条目
```

### 精确短语匹配

用双引号包裹短语，进行精确匹配：

```
"hello world"     → 包含完整短语 "hello world" 的条目
```

### 排除关键词

在关键词前加 `-` 可排除包含该词的条目：

```
hello -world      → 包含 "hello" 但不含 "world" 的条目
code -test -debug → 包含 "code"，排除 "test" 和 "debug"
```

### 组合示例

```
react "state management" -redux
```

查找包含 "react" 和精确短语 "state management"，但不含 "redux" 的条目。

## 正则搜索

用正斜杠包裹正则表达式模式：

```
/\d{4}-\d{2}-\d{2}/     → 匹配日期格式，如 2024-01-15
/https?:\/\/\S+/         → 匹配 URL
/TODO|FIXME|HACK/        → 匹配任意一个标记
/^import\s+/             → 匹配以 "import" 开头的行
```

正则模式激活时，搜索栏左侧图标会变为 `.*` 作为视觉提示。

> **注意：** 正则搜索会扫描所有字段但不使用 FTS 索引，在数据量非常大时可能略慢。

## 拼音搜索

输入拼音即可匹配中文内容：

```
nihao        → 匹配 "你好"
nh           → 匹配 "你好"（首字母）
jiemu        → 匹配 "节目"
```

全拼和首字母拼音会在条目录入时自动建立索引。

## 时间范围筛选

将鼠标悬停在搜索栏上，右侧会出现日历图标。点击可按时间范围筛选：

| 选项 | 范围 |
|------|------|
| 全部时间 | 不限时间 |
| 今天 | 当天的条目 |
| 最近 3 天 | 最近 3 天 |
| 本周 | 最近 7 天 |
| 本月 | 最近 30 天 |
| 最近 3 个月 | 最近 90 天 |

时间筛选激活时，日历图标会保持高亮并显示小圆点提示 — 即使鼠标移开搜索栏也可见。

时间筛选可与关键词搜索组合使用。

## 搜索历史

将鼠标悬停在搜索栏上（输入框为空时），会显示最近的搜索记录：

- 点击历史记录项可重新执行该搜索
- 点击条目右侧的 **X** 按钮可删除单条记录
- 点击 **清除全部历史** 可清空所有记录

最多保存 50 条搜索历史。重复搜索会更新已有记录而非新建。

## 图片 OCR 搜索

当自动 OCR 开启时（设置 > OCR > 触发模式 = "剪贴板复制时"），复制的图片会自动提取文字并建立搜索索引。这意味着你可以搜索图片中出现的文字。

当图片通过 OCR 文字匹配到搜索关键词时，匹配的文字片段会显示在图片缩略图下方，并高亮关键词。

## 搜索结果高亮

搜索时，匹配的关键词会在结果列表中高亮显示：

- **文本预览** — 条目卡片中高亮匹配词，显示匹配位置附近的上下文
- **AI 摘要** — 摘要区块中高亮匹配词
- **文件名** — 文件路径条目中高亮匹配词
- **图片 OCR 文字** — 图片缩略图下方显示匹配的 OCR 文字

普通关键词搜索和正则搜索模式均支持高亮。文本预览具有上下文感知能力 — 当匹配词不在文本开头时，预览会自动滚动到匹配位置附近显示上下文。

## 使用技巧

- 搜索**不区分大小写**，普通搜索和正则搜索均如此
- 关键词搜索结果按**相关性排序**（FTS5 排名）
- 正则搜索结果按**创建时间排序**
- 搜索输入有 200ms 防抖，避免频繁查询
- 置顶的条目始终排在最前，不受搜索排名影响
